# Design Rules

This folder is intentionally empty in the shared baseline harness.

Design-rule cards in this folder should be generated or curated from project
source intake, such as `wiki/raw/`, Docling conversion output, PDF spec
evidence, web research notes, vendor app notes, or engineer-approved internal
references.

Do not seed this folder with strong SI/PI design rules that look like final
engineering knowledge unless their source lineage is explicit. Baseline
guidance belongs in `CODEX.md`, `.codex/skills/`, docs, or templates. Generated
cards must follow `wiki/schema.md` and should preserve `source_ids`,
`source_tier`, `outputs_to`, `confidence`, and `missing_information`.
