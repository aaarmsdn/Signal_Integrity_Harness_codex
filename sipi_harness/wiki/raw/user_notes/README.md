# User Notes

Store engineer-written notes here before promoting them into typed wiki cards.

Good notes include:

- design context
- observed failure mode
- practical rule or caveat
- applicability conditions
- uncertainty or missing evidence
- suggested validation method

These notes are not compliance evidence by themselves. If a numeric limit is
needed, link it to a reviewed tier-0 source in `extracted_evidence/`.
