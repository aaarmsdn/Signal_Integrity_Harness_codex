# Source Cards

This directory is intentionally empty in the shared repository baseline.

Source cards should be generated or curated from actual source registrations,
for example governing specifications, user-approved datasheets, app notes,
papers, internal notes, or Docling/PDF evidence bundles. Baseline policy about
where to place raw files belongs in `README.md`, `CODEX.md`,
`README_AGENT.md`, and `sipi_harness/docs/workflow.md`.

A generated source card must record source tier, access rights, version/path,
allowed usage, extraction status, and whether it can support compliance
thresholds or only engineering rationale.
